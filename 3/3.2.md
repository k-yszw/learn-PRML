第3.2節「バイアスーバリアンス分解」は、モデルの複雑さと予測精度の関係を、頻度主義的な視点から解明する非常に重要なセクションです。この議論は、最尤推定がなぜ過学習を起こすのか、そして適切なモデルの複雑さをどう捉えるべきかという問いに、**「バイアス」**と**「バリアンス」**という2つの概念を切り口として答えるものです。

以下、資料の流れに沿って、数式の導出と図の解釈を交えて解説します。

### 1. 期待損失の分解（背景）

まず、第1.5.5節での議論を振り返ります。ある入力 $\mathbf{x}$ に対して目標変数 $t$ を予測する際、二乗損失関数を用いた場合の期待損失 $E[L]$ は、次のように分解されました（式 3.37）。 $$E[L] = \int {y(\mathbf{x}) - h(\mathbf{x})}^2 p(\mathbf{x}) d\mathbf{x} + \int \int {h(\mathbf{x}) - t}^2 p(\mathbf{x}, t) d\mathbf{x} dt$$

- **$h(\mathbf{x}) = \mathbb{E}[t|\mathbf{x}]$**: 理論上の最適な予測関数（回帰関数）です。
- **第2項（ノイズ）**: データに含まれる本質的なノイズであり、どのようなモデルを用いても削減できない最小の誤差です。
- **第1項**: 私たちが学習によって得た関数 $y(\mathbf{x})$ が、真の関数 $h(\mathbf{x})$ からどれだけ離れているかを示します。バイアスーバリアンス分解の焦点はこの第1項にあります。

### 2. バイアスーバリアンス分解の導出

学習結果 $y(\mathbf{x}; \mathcal{D})$ は、使用した特定の訓練データセット $\mathcal{D}$ に依存します。頻度主義的な立場では、**「もし、別のデータセット $\mathcal{D}$ を何度も観測し直して学習を繰り返したら、平均的にどのような誤差が生じるか」**を考えます。

数式(3.40)において、真の関数 $h(\mathbf{x})$ と学習された関数 $y(\mathbf{x}; \mathcal{D})$ の差の期待値を、以下の手順で展開します。

1. **「平均的な学習結果」 $\mathbb{E}_{\mathcal{D}}[y(\mathbf{x}; \mathcal{D})]$ を足して引く**: ${y(\mathbf{x}; \mathcal{D}) - h(\mathbf{x})}^2 = {y(\mathbf{x}; \mathcal{D}) - \mathbb{E}_{\mathcal{D}}[y(\mathbf{x}; \mathcal{D})] + \mathbb{E}_{\mathcal{D}}[y(\mathbf{x}; \mathcal{D})] - h(\mathbf{x})}^2$
2. **期待値をとる**: 上式を展開してデータセット $\mathcal{D}$ に関する期待値をとると、交差項が $0$ になり、次の2つの項が残ります。 $$\mathbb{E}_{\mathcal{D}}[{y(\mathbf{x}; \mathcal{D}) - h(\mathbf{x})}^2] = \underbrace{(\mathbb{E}_{\mathcal{D}}[y(\mathbf{x}; \mathcal{D})] - h(\mathbf{x}))^2}_{(\text{bias})^2} + \underbrace{\mathbb{E}_{\mathcal{D}}[{y(\mathbf{x}; \mathcal{D}) - \mathbb{E}_{\mathcal{D}}[y(\mathbf{x}; \mathcal{D})]}^2]}_{\text{variance}}$$

#### 各項の数学的意味

- **バイアス (bias)^2**: 多くのデータセットにわたる**予測の平均値が、真の関数 $h(\mathbf{x})$ からどれだけずれているか**を表します。モデルが単純すぎて真の構造を捉えきれていない場合に大きくなります。
- **バリアンス (variance)**: 個々のデータセットから得られる**予測結果が、その平均値の周りでどれだけ変動するか**を表します。モデルが複雑すぎて、個々のデータセットに含まれるランダムなノイズに敏感に反応しすぎている（過学習している）場合に大きくなります。

最終的に、期待損失は「(バイアス)^2 + バリアンス + ノイズ」の3つの和で構成されることが示されます（式 3.41）。

### 3. 図の解釈：トレードオフの視覚化

この「バイアス」と「バリアンス」は、一方を小さくしようとするともう一方が大きくなるという**トレードオフの関係**にあります。これを直感的に示しているのが **図3.5** と **図3.6** です。

#### 図3.5：正則化係数 $\lambda$ による変化

24個のガウス基底関数を用いたモデルに対し、100種類のデータセットで学習を行った例です。

- **$\ln \lambda = 2.6$ (正則化が強い / モデルが硬い)**: 左側の図では、100個の曲線がどれも似たような形をしています（**低バリアンス**）。しかし、右側の平均曲線（赤実線）は真の曲線（青破線）から大きくずれています（**高バイアス**）。
- **$\ln \lambda = -2.4$ (正則化が弱い / モデルが柔らかい)**: 左側の図では、曲線が激しく波打ち、データセットごとにバラバラな形をしています（**高バリアンス**）。一方で、右側の平均曲線は真の曲線に非常によく一致しています（**低バイアス**）。

#### 図3.6：誤差の合計と最適なバランス

横軸をモデルの複雑さ（$\ln \lambda$ の減少方向）、縦軸を誤差としてプロットした図です。

- モデルを複雑にすると、(バイアス)^2 は減少しますが、バリアンスは急上昇します。
- **テスト誤差（全誤差）**は、これら2つの和にノイズを加えたものになるため、**ある中間的な複雑さで最小値をとる「U字型」の曲線**を描きます。これが、私たちが目指すべき最適なモデルの複雑さです。

### 4. まとめと批判的見解

バイアスーバリアンス分解は、過学習の本質を理解する上で非常に強力な道具です。しかし、資料の最後（149ページ）では、この頻度主義的なアプローチの限界についても触れられています。

- **現実的な制約**: バイアスとバリアンスを計算するには、多数のデータセットを平均化する必要があります。しかし、現実にはデータセットは1つしか手に入らないことが多く、この分解を直接計算することは困難です。
- **ベイズ的な解決**: これに対して、次節（3.3節）で扱う**ベイズ線形回帰**では、複数のデータセットを想定する代わりに、単一のデータセットに基づいてパラメータの「不確実性（分布）」を扱うことで、より自然にこの問題に対処します。

バイアスーバリアンス分解は、モデルが直面する「真の構造への適合（バイアス）」と「データのノイズへの過適合（バリアンス）」の間の葛藤を、数学的に厳密に定義した概念であると言えます。
