第3.5.1節「エビデンス関数の評価」では、超パラメータ（$\alpha, \beta$）を決定するために必要な**周辺尤度（エビデンス関数） $p(\mathbf{t}|\alpha, \beta)$** を、数理的にどのように導出し、それがどのような意味を持つのかが詳しく述べられています。

資料の流れに沿って、数式の導出と図の解釈を解説します。

### 1. エビデンス関数の定義と出発点 (式 3.77)

エビデンス関数 $p(\mathbf{t}|\alpha, \beta)$ は、パラメータ $\mathbf{w}$ を周辺化（積分）することで得られます。 $$p(\mathbf{t}|\alpha, \beta) = \int p(\mathbf{t}|\mathbf{w}, \beta) p(\mathbf{w}|\alpha) d\mathbf{w} \quad (3.77)$$ この積分は、尤度関数（ガウス分布）と事前分布（ガウス分布）の積を全パラメータ空間で合計することを意味します。第2.3.3節の線形ガウスモデルの結果を利用して解くこともできますが、ここでは指数部分を整理する手法で導出が進められます。

### 2. 指数部分の整理と誤差関数 (式 3.78, 3.79)

尤度と事前分布を掛け合わせると、指数部分には正規化された二乗和誤差関数と二次の正則化項が合わさった形が現れます。これを次のように定義します。 $$p(\mathbf{t}|\alpha, \beta) = \left( \frac{\beta}{2\pi} \right)^{N/2} \left( \frac{\alpha}{2\pi} \right)^{M/2} \int \exp { -E(\mathbf{w}) } d\mathbf{w} \quad (3.78)$$ ここで、$E(\mathbf{w})$ は以下の通りです。 $$E(\mathbf{w}) = \beta E_D(\mathbf{w}) + \alpha E_W(\mathbf{w}) = \frac{\beta}{2} |\mathbf{t} - \boldsymbol{\Phi}\mathbf{w}|^2 + \frac{\alpha}{2} \mathbf{w}^T\mathbf{w} \quad (3.79)$$

### 3. $\mathbf{w}$ に関する積分の実行 (式 3.80 〜 3.85)

この積分を実行するために、$E(\mathbf{w})$ をその最小値（すなわち事後平均 $\mathbf{m}_N$）の周りで平方完成します。 $$E(\mathbf{w}) = E(\mathbf{m}_N) + \frac{1}{2} (\mathbf{w} - \mathbf{m}_N)^T \mathbf{A} (\mathbf{w} - \mathbf{m}_N) \quad (3.80)$$ ここで、**$\mathbf{A}$ は誤差関数の2階導関数行列（ヘッセ行列）**であり、次のように定義されます。 $$\mathbf{A} = \nabla\nabla E(\mathbf{w}) = \alpha \mathbf{I} + \beta \boldsymbol{\Phi}^T \boldsymbol{\Phi} \quad (3.81)$$ この $\mathbf{A}$ を用いてガウス積分の公式を適用すると、$\mathbf{w}$ に関する積分は次のような簡潔な形で評価できます。 $$\int \exp { -E(\mathbf{w}) } d\mathbf{w} = \exp { -E(\mathbf{m}_N) } (2\pi)^{M/2} |\mathbf{A}|^{-1/2} \quad (3.85)$$

### 4. 対数エビデンス関数の最終形 (式 3.86)

最後に、これら全てをまとめて対数をとることで、超パラメータ $\alpha, \beta$ を最適化するための目標関数である**対数エビデンス関数**が導かれます。 $$\ln p(\mathbf{t}|\alpha, \beta) = \frac{M}{2} \ln \alpha + \frac{N}{2} \ln \beta - E(\mathbf{m}_N) - \frac{1}{2} \ln |\mathbf{A}| - \frac{N}{2} \ln(2\pi) \quad (3.86)$$

**数式の意味：** この式は、モデルの良さを以下の要素のバランスとして評価しています。

- **$E(\mathbf{m}_N)$**: 訓練データに対するフィッティングの良さ（負の符号があるため、誤差が小さいほどエビデンスは高まる）。
- **$\frac{1}{2} \ln |\mathbf{A}|$**: モデルの複雑さ（行列 $\mathbf{A}$ の行列式）に対するペナルティ。第3.4節で議論された「オッカムの剃刀」を数理的に表現した項です。

### 5. 図3.14の解釈：モデルの次数とエビデンスの関係

資料の167ページにある **図3.14** は、多項式曲線フィッティングの例を用いて、多項式の次数 $M$ （モデルの複雑さ）とエビデンス（式 3.86）の関係をプロットしたものです。

- **$M$ が小さいとき ($M=0, 1, 2$)**: モデルが単純すぎてデータを十分に説明できない（アンダーフィッティング）ため、エビデンスの値は低くなります。
- **最適な $M$ ($M=3$)**: エビデンスが最大値をとります。これは、データの背後にある構造（この例では3次多項式付近）を最も効率的に説明できていることを示しています。
- **$M$ が大きいとき ($M > 3$)**: データへの適合度は上がりますが、モデルが複雑すぎるために「ペナルティ項」が支配的になり、エビデンスの値は再び減少していきます。

### 結論

3.5.1節の内容は、**「エビデンス関数を最大化することは、データへの適合度とモデルの簡潔さの最良のバランスを、交差確認なしに数学的に見つけ出す作業である」**ということを、具体的な計算プロセスと図表を通じて証明しています。この後、資料はこれをさらに進めて、$\alpha$ と $\beta$ の具体的な更新式を導く議論へと続いていきます。
