第3.5.2節「エビデンス関数の最大化」では、前節で導出した対数エビデンス関数（式 3.86）を最大化することで、**超パラメータ $\alpha$ と $\beta$ をデータから直接決定するための具体的な更新式とその数学的意味**が解説されています。

このプロセスは、モデルの複雑さを「有効パラメータ数」という概念で捉え直す非常に重要なステップです。

---

### 1. 数理的な導出：停留条件の計算

エビデンス関数 $p(\mathbf{t}|\alpha, \beta)$ を最大化するために、まず $\alpha$ および $\beta$ に関する偏微分を $0$ と置くことで停留点を求めます。

#### $\alpha$ に関する最適化 (式 3.87 〜 3.92)

対数エビデンス関数（式 3.86）には、行列 $\mathbf{A} = \alpha \mathbf{I} + \beta \boldsymbol{\Phi}^T \boldsymbol{\Phi}$ の行列式 $\ln |\mathbf{A}|$ が含まれています。これを計算しやすくするため、行列 $\beta \boldsymbol{\Phi}^T \boldsymbol{\Phi}$ の固有値方程式を考えます。 $$(\beta \boldsymbol{\Phi}^T \boldsymbol{\Phi}) \mathbf{u}_i = \lambda_i \mathbf{u}_i \quad (3.87)$$ これにより、行列 $\mathbf{A}$ の固有値は $\alpha + \lambda_i$ となり、$\ln |\mathbf{A}| = \sum_i \ln (\alpha + \lambda_i)$ と書けます。

これを $\alpha$ で微分して整理すると、$\alpha$ の最適解は次の形になります。 $$\alpha = \frac{\gamma}{\mathbf{m}_N^T \mathbf{m}_N} \quad (3.92)$$ ここで登場する **$\gamma$** は次のように定義され、本節の核心となる量です。 $$\gamma = \sum_{i=1}^M \frac{\lambda_i}{\alpha + \lambda_i} \quad (3.91)$$

#### $\beta$ に関する最適化 (式 3.93 〜 3.95)

同様に、対数エビデンス関数を $\beta$ で微分して整理すると、精度の最適解（分散の逆数）が得られます。 $$\frac{1}{\beta} = \frac{1}{N - \gamma} \sum_{n=1}^N {t_n - \mathbf{m}_N^T \boldsymbol{\phi}(\mathbf{x}_n)}^2 \quad (3.95)$$

---

### 2. 「有効パラメータ数 $\gamma$」の幾何学的意味 (図3.15の解釈)

導出された更新式に含まれる **$\gamma$** は、モデルに含まれる全パラメータ数 $M$ のうち、**「データによってその値が実質的に決定された（well-determined）」パラメータの数**を表しています。

**図3.15** は、この概念をパラメータ空間（ここでは $w_1, w_2$ の2次元）の等高線で視覚化しています。

- **尤度関数（実線）と事前分布（破線）:**
    - 尤度関数の広がり（曲率）は固有値 $\lambda_i$ によって決まり、事前分布の広がりは $\alpha$ によって決まります。
- **方向による違い:**
    - **$w_1$ の方向（$\lambda_1 \gg \alpha$）**: 尤度関数のピークが非常に鋭く、事前分布よりも支配的です。このとき比率 $\lambda_i / (\alpha + \lambda_i)$ は $1$ に近くなり、このパラメータは**「データによってしっかり決まっている」**とみなされ、$\gamma$ のカウントに $1$ 寄与します。
    - **$w_2$ の方向（$\lambda_2 \ll \alpha$）**: 尤度関数の変化が緩やかで、事前分布の方が支配的です。このとき比率は $0$ に近くなり、このパラメータはデータからは決まらず**「事前分布に引きずられている」**とみなされ、$\gamma$ にはほとんど寄与しません。
- **結論:** $\gamma$ はこれら $M$ 個の方向の寄与を合計したものであり、常に $0 \le \gamma \le M$ の範囲に収まります。

---

### 3. 話の流れ：逐次的な再推定アルゴリズム

ここで得られた $\alpha$ と $\beta$ の解（式 3.92, 3.95）は、右辺の $\gamma$ や事後平均 $\mathbf{m}_N$ 自体が $\alpha, \beta$ に依存しているため、直接一度に解くことはできません。

そのため、資料では以下の**反復的なアルゴリズム**が提案されています。

1. $\alpha, \beta$ の初期値を適当に決める。
2. 現在の $\alpha, \beta$ を用いて、事後平均 $\mathbf{m}_N$ と固有値 $\lambda_i$（および $\gamma$）を計算する。
3. 式(3.92)と(3.95)を使って、新しい $\alpha$ と $\beta$ を再推定する。
4. 値が収束するまで 2 と 3 を繰り返す。

---

### 4. 最尤推定との比較（バイアスの補正）

$\beta$ の更新式(3.95)に注目すると、分母が $N$ ではなく **$N - \gamma$** になっている点が非常に重要です。

- **最尤推定（式 1.63）**: 分母は $N$ でした。これは、モデルがデータに過剰適合し、ノイズの分散を過小評価してしまうバイアスがありました。
- **エビデンス近似**: データの総数 $N$ から、モデルのフィッティングに使われてしまった「有効な自由度 $\gamma$」を差し引くことで、**分散の過小評価をベイズ的に正しく補正しています**。これは、第1.2.4節で学んだ「ガウス分布の分散の最尤推定が持つバイアス」を、より複雑な回帰モデルに一般化したものと解釈できます。

### まとめ

第3.5.2節は、超パラメータの決定という問題を、**「データの情報量（尤度の曲率）」と「事前知識の重み（$\alpha$）」のせめぎ合い**として数理的に定式化しています。**図3.15** が示す通り、データが十分にある方向のパラメータのみを効率的に使い、そうでない方向は事前分布で抑え込むという、ベイズ推論の持つ**「モデルの複雑さを自動調整するメカニズム」**を、$\gamma$ という具体的な指標で明らかにしたセクションです。
