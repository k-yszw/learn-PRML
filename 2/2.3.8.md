第2.3.8節「周期変数」について、資料の流れに沿って、専門知識がなくてもイメージしやすいように解説します。

### 1. なぜ「周期変数」には特別な扱いが必要なのか？

通常、実数値（気温や身長など）は直線の上の値として扱いますが、世の中には **「一周して戻ってくる」変数** があります。

- **例：** 風向、日付、時刻（24時間で0時に戻る）など。

これらの変数を普通の数として扱うと、大きな問題が起こります。例えば、方位磁針で **「$1^\circ$（北）」と「$359^\circ$（北）」** の平均を計算しようとすると、普通の計算（算術平均）では $(1+359)/2 = 180^\circ$（南！）になってしまいます。実際には北に近い方向を指しているはずなのに、真逆の結果になってしまうのです。これを解決するのが「周期変数」の統計学です。

### 2. 角度を「円の上の点」として捉える（図 2.17）

この問題を避けるために、角度 $\theta$ を単なる数字ではなく、**「単位円（半径1の円）の上の座標」** として表現します。

- **数式 (2.167):** 各データを2次元のベクトル $\mathbf{x}_n = (\cos \theta_n, \sin \theta_n)^T$ として扱います。
- **平均の求め方:** これらのベクトルを全て足して平均をとります。その結果得られた平均ベクトル $\bar{\mathbf{x}}$ が指す方向が、データの「平均的な向き」になります。
- **数式 (2.169):** 平均の角度 $\bar{\theta}$ は、以下の式で計算できます。 $$\bar{\theta} = \tan^{-1} \left( \frac{\sum \sin \theta_n}{\sum \cos \theta_n} \right)$$ この方法なら、先ほどの $1^\circ$ と $359^\circ$ の平均も、正しく北（$0^\circ$）付近を指すようになります。

### 3. フォン・ミーゼス分布：円の上のガウス分布（図 2.18）

ガウス分布を円の上に「巻き付けた」ような分布が、**フォン・ミーゼス分布（von Mises distribution）** です。これは、2次元のガウス分布（図 2.18の赤い等高線）を考え、それを半径1の円のラインだけで切り取ったものと解釈できます。

- **分布の式 (2.179):** $$p(\theta | \theta_0, m) = \frac{1}{2\pi I_0(m)} \exp { m \cos(\theta - \theta_0) }$$ ここで重要なパラメータは以下の2つです。
    1. **$\theta_0$ (平均方向):** 分布の山がどこにあるか（中心）を示します。
    2. **$m$ (集中度パラメータ):** 分布がどれだけ中心に集中しているかを示します。ガウス分布でいう「分散の逆数（精度）」に近い役割です。

### 4. 分布の形とベッセル関数（図 2.19, 2.20）

- **図 2.19:** $m$ の値による分布の変化が描かれています。
    - **$m$ が大きいとき:** 分布は鋭い山になり、ガウス分布に非常に似た形になります。
    - **$m=0$ のとき:** 全ての方向に均等に広がる（一様分布）になります。
- **ベッセル関数 $I_0(m)$ (式 2.180):** 数式にあるこの項は、分布全体の面積を1にするための調整役です。**図 2.20（左）** にあるように、$m$ が大きくなるほど急激に値が大きくなります,。

### 5. パラメータの推定（最尤推定）

データから最もらしい $\theta_0$ と $m$ を見つける方法です。

- **$\theta_0$ の推定 (式 2.184):** 前述の「ベクトルの平均」から求めた角度がそのまま使われます。
- **$m$ の推定 (式 2.185, 2.186):** **図 2.20（右）** にある $A(m)$ という関数を使います。データの平均ベクトルの長さ（$\bar{r}$）を計算し、それが $A(m)$ と一致するような $m$ を探すことで、データの「集中具合」を推定します。

### まとめ

周期変数の扱いにおいて重要なのは、**「直線を円に丸めて考える」** ことです。これにより、端と端がつながっているデータに対しても、不自然なジャンプを起こさずに統計的な処理（フォン・ミーゼス分布によるモデル化）が可能になります。
